# Моделирование и комплексирование БИНС/СНС для БКТС

## Содержание работы

1. **Моделирование БИНС.**

   Генерация ИИМ и ГНСС в [Aceinna GNSS-INS-SIM](https://github.com/Aceinna/gnss-ins-sim). Ориентация определяется фильтром Маджвиком, Махони. Механизация в NED.

2. **Комплексирование ИНС/СНС.** 

   Слабосвязанная схема комплексирования через EKF, UKF и оптимизацию на факторном графе (FGO, на базе [GTSAM](https://gtsam.org/)).

3. **Сравнение методов**


4. **Обработка реальных данных.**

   Предобработка записей с самосвала (2 ГНСС-приёмника 10 Гц, БИНС 400 Гц), ручная разметка на сценарии и прогон тех же фильтров на реальных сегментах.


## Условия

- Все измерения в системе координат NED (при стоянке +1 g по оси Oz). 
- Линейные ускорения `accel` в м/с², гироскоп `gyro` в рад/с
- Линейные скорости с ГНСС в NED, переводится из км/ч в м/с.
- Инициализация без эталона: `init_r = 0`, `init_v` — первая GPS-скорость, `q0 = [1,0,0,0]` (крен/тангаж фильтр оценивает сам).
- Метрики не считаются — у реальных записей нет истинной траектории; методы сравниваются между собой и по невязкам на RTK-участках. При моделировании метки выводятся

### Логика 

| Группа | Что делает | 
|------|-------------|
| `simulation` | *Генерация* данных: профили, прогон симулятора, вибрация, загрузка | 
| `preprocessing` | *Реальные данные*: предобработка .csv, разметка, нарезка, подготовка ко входу в фильтры |
| `navigation` | *Оценка* состояния: счисление, EKF/UKF/FGO | 
| `analysis` | *Анализ результатов*: перевод в NED, метрики, графики |
